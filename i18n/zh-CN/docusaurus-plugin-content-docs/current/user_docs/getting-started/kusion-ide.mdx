# 使用 VS Code Kusion

[VS Code Kusion extension](https://marketplace.visualstudio.com/items?itemName=KusionStack.kusion) 提供了很方便的操作来将 KCL 配置带到到云端。

## 开始之前

您可以从云 IDE上 [GitHub Codespaces](https://github.com/features/codespaces) 的 VS Code Kusion 开始，或者您可以将它安装在桌面版 VS Code 上。

    {label: '桌面端 VS Code', value: 'desktop'},

<TabItem value="cloudide">

恭喜！ 🎉🎉 您无需设置环境！ 您可以直接[使用内置的 Kusion 创建工作区](https://github.com/codespaces/new?hide_repo_select=true&ref=main&repo=488867056&machine=standardLinux32gb&devcontainer_path=.devcontainer.json)。

</TabItem>
<TabItem value="desktop">

**步骤 1.** 在您的系统上安装 Kusion。 请参考[安装 Kusion](/docs/user_docs/getting-started/install)。

**步骤 2.** 为 Visual Studio Code 安装 Kusion 扩展。 此扩展需要 VS Code 1.68+版本。

**步骤 3.** 您需要有一个 Kubernetes 集群并将 kubeConfig 文件放在 `~/.kube/config`下。

</TabItem>
</Tabs>

## 开始交付您的第一个应用程序

下面是一个示例，让您快速开始使用 VS Code Kusion Extension 将 [code city](https://wettel.github.io/codecity.html) 应用程序交付到云端。 所有步骤都可以在 VS Code 中的 Kusion `Getting started with Kusion` 中交互式体验。

要查找 Kusion walkthrough ：打开 `Command Palatte` > 键入并选择 `Welcome: Open Walkthrough...` > 然后键入并选择 `kusion`。

### 1. 建模: 定义你的模型

### 2. 配置: 新建 Kusion 项目

您可以从原型快速创建一个新的 kusion 项目。 为此，请单击 walkthrough 中的 `Create Kusion Project` 按钮（或者，在命令面板中键入 `Kusion：Create`），然后选择一个项目模板（例如，使用 `code-city` 模板，您可以部署一个 3D 城市可视化的应用程序）。

![](/img/docs/user_docs/getting-started/create-project.gif)

### 3. 预览：YAML 表示

现在让我们预览我们之前创建的配置的 YAML 表示，方法是单击数据预览按钮或键入并选择 `Kusion：Open Data Preview To the Side`。

![](/img/docs/user_docs/getting-started/data-preview.gif)


### 4.运行时差异和上线
    
要查看当前运行时差异，您可以右键单击配置主文件并选择 `Diff with Runtime and Apply` 以打开运行时差异页面。

然后确认差异并使更改上线。

![](/img/docs/user_docs/getting-started/config-diff-apply.gif)
